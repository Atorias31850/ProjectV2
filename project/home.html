ยกระดับทักษะการพัฒนาระบบ
ด้วย Git และ GitHub อย่างมืออาชีพ

ก้าวข้ามข้อจำกัดในการจัดการไฟล์แบบดั้งเดิม เรียนรู้ระบบควบคุมเวอร์ชัน (Version Control System) ตั้งแต่ระดับพื้นฐานจนถึงสถาปัตยกรรมการทำงานร่วมกัน (Gitflow) มาตรฐานอุตสาหกรรมซอฟต์แวร์

▶ เริ่มต้นเข้าสู่บทเรียน

เนื้อหาบทเรียน (Course Syllabus)

บทที่ 1: ปัญหาของการจัดการไฟล์แบบเดิม

ศึกษาข้อจำกัดของการจัดการไฟล์โดยไม่ใช้ระบบควบคุมเวอร์ชัน และทำความเข้าใจความสำคัญของ Git ในการแก้ปัญหาการสูญหายของข้อมูลและการทำงานซ้ำซ้อน

~ % rm -rf Project_Final_v99

อ่านเนื้อหาบทที่ 1 คลิกที่นี่

บทที่ 2: โครงสร้างและการทำงานของ Git

ทำความเข้าใจสถาปัตยกรรมของ Git (Working Directory, Staging Area, Repository) และการใช้คำสั่งพื้นฐานในการเริ่มต้นโครงการและบันทึกการเปลี่ยนแปลง (Commit)

~ % git add .
~ % git commit -m "feat: login system"

บทที่ 3: การจัดการสาขา (Branching)

ศึกษาวิธีการแยกสาขา (Branch) เพื่อพัฒนาส่วนย่อยของโปรแกรม (Feature) โดยไม่กระทบกับระบบหลัก และหลักการทำงานร่วมกันเป็นทีมอย่างมีประสิทธิภาพ

~ % git branch feature-login
~ % git checkout feature-login

บทที่ 4: การรวมรหัสต้นฉบับ (Merge & PR)

เรียนรู้กระบวนการ Pull Request เพื่อตรวจสอบความถูกต้องของรหัสต้นฉบับก่อนทำการรวม (Merge) และวิธีการแก้ไขข้อขัดแย้งของข้อมูล (Merge Conflict)

~ % git merge feature-login

บทที่ 5: การประยุกต์ใช้แพลตฟอร์ม GitHub

ศึกษาการเชื่อมต่อพื้นที่เก็บข้อมูลในเครื่อง (Local Repository) เข้ากับระบบคลาวด์ (Remote Repository) เพื่อการสำรองข้อมูลและการนำเสนอผลงาน (Portfolio)

~ % git remote add origin [URL]

บทที่ 6: ภาคปฏิบัติและสรุปคำสั่ง (Workshop)

ฝึกปฏิบัติการใช้งานจริงผ่านสถานการณ์จำลอง ตั้งแต่การนำเข้า (Pull) และส่งออกข้อมูล (Push) พร้อมตารางสรุปคำสั่งที่จำเป็นสำหรับการปฏิบัติงาน (Cheat Sheet)

~ % git push -u origin main
~ % git pull origin main
